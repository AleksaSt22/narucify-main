<analysis>**original_problem_statement:**
The user wants to build a web application named DM → ORDER SYSTEM. The system is designed for individuals who sell products through direct messages (on platforms like Instagram, WhatsApp) but lack a formal online store.

**PRODUCT REQUIREMENTS:**
- **Core Concept:** The system should replace the chaotic nature of DM orders. A seller can select one or more products, generate a unique order link, and send it to a customer. The customer uses this link to view the specific products, enter their shipping information, and finalize the order.
- **No Payment Processing:** The system will not handle any financial transactions (no credit cards, Stripe, etc.). It's purely for organizing orders, customers, and products.
- **Seller Flow:**
    1. Register/Login to an admin panel.
    2. Add products (name, image, price, stock quantity).
    3. Create an order by selecting products for a customer.
    4. Copy the generated unique order link and send it via DM.
    5. View new orders, their statuses, customer data, and updated stock levels in the admin panel.
- **Customer Flow:**
    1. Click the received link to open a product-specific order page.
    2. View product details (image, name, price).
    3. Enter personal information (name, phone, address, optional email).
    4. Choose a payment method (e.g., cash on delivery).
    5. Submit the order and receive a confirmation with an Order ID and a permanent link to track the order's status.
- **MVP Functionality:**
    - **Admin Panel:** User authentication, Product CRUD, Order list with statuses (New, Confirmed, Shipped, etc.), automatic stock updates, and customer data overview.
    - **Public Pages:** A non-navigable product order page, an order form, and an order confirmation page.
- **Post-MVP (Locked Features):** The system should include modules for Analytics, Finance, Customer Management, and Email Marketing. These features are initially locked and can be activated or purchased by the user.
- **Super Admin:** A separate panel for the platform owner to view all registered sellers and manage their access to the locked premium features.
- **Monetization & Premium Features:**
    - Implement a pricing model where users can unlock features individually (e.g., Analytics - €5.99) or as a bundled PRO Plan (€13.99).
    - The UI for locked features should be visually appealing, with animations and modals that explain the value proposition of each feature.
    - **PRO features requested:** Referral program, a mini-shop page for sellers, priority support, custom branding (logo upload, watermark removal), and gamification (badges).

**User's preferred language**: The user communicates in **Serbian**. The next agent MUST respond in Serbian.
